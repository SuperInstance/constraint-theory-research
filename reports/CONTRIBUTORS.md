## How to Contribute

We welcome contributions from everyone! See [CONTRIBUTING.md](CONTRIBUTING.md) for details.

### Ways to Contribute

- **Code:** Bug fixes, features, performance improvements
- **Mathematics:** Theorems, proofs, algorithms, optimizations
- **Documentation:** Guides, tutorials, API docs
- **Tests:** Unit tests, property-based tests, benchmarks
- **Design:** Visualizations, diagrams, examples
- **Community:** Support, advocacy, outreach, research

### Getting Started

1. Read [CONTRIBUTING.md](CONTRIBUTING.md)
2. Check [good first issues](https://github.com/SuperInstance/constrainttheory/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)
3. Join [Discussions](https://github.com/SuperInstance/constrainttheory/discussions)
4. Start contributing!

**Bronze Contributor** (1-4 contributions)
- Listed in contributors
- Thanked in release notes

- Featured contributor status
- Invited to research discussions

**Gold Contributor** (10+ contributions)
- Hall of fame listing
- Permanent recognition
- Annual contributor gift

**Research Contributor** (Significant mathematical contributions)
- Co-authorship on papers
- Academic recognition
- Invited to seminars

## Become a Maintainer

We're always looking for new maintainers!

**Requirements:**
- Consistent, high-quality contributions
- Active community participation
- Technical or mathematical expertise
- Endorsed by existing maintainers

**Benefits:**
- Full repository access
- Vote in decisions
- Represent the project
- Shape the future
- Co-author papers

**Process:**
1. Contribute consistently
2. Review pull requests
3. Help with releases
4. Be nominated by existing maintainers
